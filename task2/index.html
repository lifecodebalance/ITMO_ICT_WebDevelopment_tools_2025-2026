    Задача 2: парсинг страниц + запись в БД

    Что требовалось

    Для каждого подхода (threading, multiprocessing, asyncio) реализовать parse_and_save(url), которая:

    1. Загружает HTML по URL;
    2. Извлекает заголовок страницы (<title>);
    3. Сохраняет данные в БД;
    4. Печатает результат в консоль.

    Почему выбраны таблицы ЛР1

    Вместо создания новой базы используется база ЛР1 (по DB_TIME). Данные парсинга сохраняются в существующую модель:

    • создаётся/используется пользователь lab2_parser;
    • создаётся категория вида Parsed pages (approach);
    • каждая спарсенная страница добавляется как запись в таблицу task.

    Так мы действительно «заполняем БД ЛР1 данными парсинга».

    Асинхронное подключение к БД

    Слой БД сделан в lr2.task2.db_async на SQLAlchemy Async Engine.

    Важно:

    • URL из DB_TIME автоматически переводится в async-диалект:
    • postgresql+psycopg:// -> postgresql+asyncpg://
    • sqlite:// -> sqlite+aiosqlite://
    • используется NullPool, чтобы избежать проблем при запуске из разных event loop/потоков/процессов;
    • все записи в БД идут через async-функцию save_page_as_task(...).

    Реализации

    threading_parser

    • HTTP-запросы: requests;
    • парсинг HTML: BeautifulSoup;
    • запись в БД: async-функция через asyncio.run(...) на каждую запись.

    multiprocessing_parser

    • HTTP-запросы: requests;
    • чанки URL раздаются процессам через Pool.map(...);
    • запись в БД также выполняется через async-функцию внутри каждого процесса.

    async_parser

    • HTTP-запросы: aiohttp;
    • одновременный запуск через asyncio.gather(...) и Semaphore;
    • запись в БД сразу async->async без перехода в sync-контекст.

    Результаты замера

    Команда:

    PYTHONPATH=src python -m lr2.task2.run_benchmark --workers 3

    Результат:

    Подход Время, сек
    threading 0.940034
    multiprocessing 1.278321
    asyncio 0.896541

    Почему так получилось

    • Парсинг — I/O-bound задача (ожидание сети), поэтому threading и asyncio обычно близки.
    • multiprocessing часто проигрывает на веб-запросах из-за тяжёлого старта процессов и дополнительной координации.
    • asyncio показывает хороший результат при большом числе одновременных запросов и грамотной неблокирующей цепочке.

    Что пишется в БД

    В таблицу task записываются:

    • title — заголовок страницы;
    • description — URL, подход и метка времени;
    • служебные поля (priority, status, created_at, и т.д.) заполняются корректными значениями для схемы ЛР1.
