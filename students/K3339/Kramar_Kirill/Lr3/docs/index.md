# Лабораторная работа 3

## Цель

Научиться упаковывать FastAPI-приложение в Docker, интегрировать парсер через HTTP и вызывать его асинхронно через очередь задач.

## Архитектура

```
Клиент
  │
  ▼
┌────────────────────────────────────────────────────────────┐
│  api  (FastAPI ЛР1, порт 8000)                             │
│                                                            │
│  POST /parse        ──────────────────────────────────────►│
│  POST /parse/async  ──► Celery queue (Redis)               │
│  GET  /parse/async/{id}                                    │
└──────────────────────────┬─────────────────────────────────┘
                           │ HTTP  POST /parse
                           ▼
                  ┌────────────────────┐
                  │  parser  (ЛР2)     │
                  │  порт 8001         │
                  └────────────────────┘
                           ▲
                           │ HTTP  POST /parse
                  ┌────────────────────┐
                  │  celery_worker     │
                  └────────────────────┘
                           ▲
                  ┌────────────────────┐
                  │  redis  (брокер)   │
                  └────────────────────┘
                           ▲  ▼ подключение к БД
                  ┌────────────────────┐
                  │  postgres          │
                  └────────────────────┘
```

## Сервисы

| Сервис | Образ / Dockerfile | Порт | Назначение |
|---|---|---|---|
| `postgres` | `postgres:16-alpine` | 5432 | БД из ЛР1 |
| `api` | `Lr1/Dockerfile` | 8000 | FastAPI-приложение |
| `parser` | `Lr2/Dockerfile.parser` | 8001 | HTTP-парсер страниц |
| `redis` | `redis:7-alpine` | 6379 | Брокер Celery |
| `celery_worker` | `Lr1/Dockerfile` | — | Воркер очереди |

## Что реализовано

- **Подзадача 1** (70%): Все сервисы упакованы в Docker и поднимаются через `docker-compose up`.
- **Подзадача 2** (100%): Эндпоинт `POST /parse` принимает URL от клиента, вызывает parser-сервис, сохраняет заголовок в БД и возвращает результат клиенту.
- **Подзадача 3** (100%): Эндпоинт `POST /parse/async` ставит задачу в очередь Celery. Воркер выполняет её в фоне. Статус проверяется через `GET /parse/async/{task_id}`.
