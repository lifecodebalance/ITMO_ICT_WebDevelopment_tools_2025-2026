# Модель данных

## Таблицы

В приложении используется 6 таблиц.

## `user`

Хранит данные пользователя:

- `id`
- `username`
- `email`
- `hashed_password`
- `created_at`

## `taskcategory`

Категория помогает группировать задачи:

- `id`
- `name`
- `color_code`
- `user_id`

## `tag`

Тег нужен для дополнительной маркировки задач:

- `id`
- `name`
- `user_id`

## `task`

Главная таблица проекта:

- `id`
- `title`
- `description`
- `deadline`
- `priority`
- `status`
- `estimated_time_minutes`
- `actual_time_minutes`
- `created_at`
- `user_id`
- `category_id`

## `tasktag`

Ассоциативная таблица между задачами и тегами:

- `task_id`
- `tag_id`
- `added_at`
- `is_primary`

Эта таблица важна для лабораторной, потому что содержит не только ссылки на сущности, но и собственные смысловые поля связи.

## `reminder`

Напоминание по задаче:

- `id`
- `remind_at`
- `is_sent`
- `task_id`

## Связи

### One-to-many

- один пользователь -> много задач;
- один пользователь -> много категорий;
- один пользователь -> много тегов;
- одна задача -> много напоминаний.

### Many-to-many

- одна задача -> много тегов;
- один тег -> много задач;
- связь реализуется через таблицу `tasktag`.
