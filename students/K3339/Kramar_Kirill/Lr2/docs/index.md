# Лабораторная работа 2

## Тема
Потоки, процессы и асинхронность в Python.

## Цель работы
Понять практическую разницу между:

- `threading` (потоки);
- `multiprocessing` (процессы);
- `asyncio` (асинхронные задачи и event loop).

И закрепить это на двух практических задачах:

1. Параллельный расчёт суммы чисел от `1` до `10_000_000_000_000`.
2. Параллельный парсинг веб-страниц с сохранением результатов в базу данных из ЛР1.

## Что сделано в проекте

Реализованы **6 отдельных программ**:

- Задача 1:
  - `lr2.task1.threading_sum`
  - `lr2.task1.multiprocessing_sum`
  - `lr2.task1.async_sum`
- Задача 2:
  - `lr2.task2.threading_parser`
  - `lr2.task2.multiprocessing_parser`
  - `lr2.task2.async_parser`

Дополнительно есть два сравнения-бенчмарка:

- `lr2.task1.run_benchmark`
- `lr2.task2.run_benchmark`

## Важный принцип реализации

Код написан максимально читаемо:

- простые функции с понятными именами;
- явные шаги обработки;
- минимально «магических» сокращений;
- без перегруза в одну строку.

## Ключевой нюанс про БД

По сообщению в потоке требовалось асинхронное подключение к БД. Это учтено:

- для сохранения данных используется `SQLAlchemy Async Engine`;
- подключение к БД берётся из `DB_TIME`, как и в ЛР1;
- записи идут в существующие таблицы ЛР1: `user`, `taskcategory`, `task`.
